## 🔍 调试技巧

### 查看 localStorage
打开浏览器开发者工具（F12），在 Console 中输入：

```javascript
// 查看管理员状态
localStorage.getItem('isAdmin')

// 手动设置为管理员（仅用于测试）
localStorage.setItem('isAdmin', 'true')
location.reload()

// 清除管理员状态
localStorage.removeItem('isAdmin')
location.reload()
```

### 查看网络请求
在 Network 标签中筛选 `admin`，可以看到：
- `POST /api/admin/verify` - 密码验证请求
- 请求体：`{password: "..."}`
- 响应：`{success: true/false, message/error: "..."}`

### 检查后端日志
后端控制台会显示：
```
Admin password configured: yes
```

## ⚠️ 常见问题排查

### 问题1：点击管理模式没有反应
**可能原因：**
- 前端服务未重启，仍在使用旧代码
- 浏览器缓存了旧的 JS 文件

**解决方案：**
```bash
# 停止前端服务（Ctrl+C）
# 重新启动
npm start

# 或者强制刷新浏览器
Ctrl + Shift + R (Windows)
Cmd + Shift + R (Mac)
```

### 问题2：密码总是验证失败
**检查步骤：**
1. 确认 `.env` 文件中 `ADMIN_PASSWORD` 的值
2. 确认后端已重启（看到 `Admin password configured: yes`）
3. 检查输入时是否有空格或大小写错误

### 问题3：刷新后管理员状态丢失
**检查步骤：**
1. 打开 F12 → Application → Local Storage
2. 确认存在键值对：`isAdmin: "true"`
3. 如果不存在，可能是浏览器禁用了本地存储

### 问题4：管理页面显示空白
**检查步骤：**
1. 打开 F12 → Console 查看错误信息
2. 确认 TemplateManager 组件是否正确导入
3. 确认路由配置是否正确
