# 导出功能测试指南

## 🔍 问题诊断

如果您遇到"导出失败: Cannot read properties of null (reading 'querySelectorAll')"错误，请按照以下步骤检查：

## ✅ 检查清单

### 1. 确认服务器运行状态

```bash
# 检查后端服务器是否运行
# 访问 http://localhost:3001
# 应该看到 "Cannot GET /" 或类似响应

# 检查前端开发服务器是否运行
# 访问 http://localhost:3000
# 应该看到课表模板编辑系统界面
```

### 2. 检查浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签
3. 点击"导出图片"按钮
4. 查看控制台输出

**正常输出应该包括：**
```
开始导出...
找到图片元素: <img>
图片已加载
容器元素: <div>
找到格子数量: X
格子 0: <div> 输入框: <input>
...
图片尺寸: XXX x XXX
开始生成canvas...
克隆文档完成
Canvas生成完成，尺寸: XXX x XXX
准备下载图片...
图片下载完成
```

**错误输出示例：**
```
开始导出...
编辑器未初始化
```
或
```
开始导出...
找到图片元素: <img>
图片已加载
容器元素: null
容器元素未找到
```

### 3. 检查模板数据

1. 确保已上传模板图片
2. 确保已标记至少一个格子
3. 确保已保存模板配置

### 4. 检查网络请求

1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 点击"导出图片"按钮
4. 查看是否有失败的请求

## 🐛 常见问题

### 问题1: "编辑器未初始化"

**原因：** editorRef 没有正确绑定到 DOM 元素

**解决方案：**
1. 刷新页面
2. 重新选择模板
3. 确保页面完全加载后再点击导出

### 问题2: "容器元素未找到"

**原因：** .template-editor 元素不存在

**解决方案：**
1. 检查模板是否正确加载
2. 检查浏览器控制台是否有其他错误
3. 尝试刷新页面

### 问题3: "未找到任何格子"

**原因：** 模板没有标记格子

**解决方案：**
1. 进入模板管理页面
2. 选择模板
3. 进入标记模式
4. 标记至少一个格子
5. 保存配置

### 问题4: 导出的图片是白板

**原因：** 图片加载失败或跨域问题

**解决方案：**
1. 确保后端服务器正常运行
2. 检查图片是否正确上传
3. 查看浏览器控制台的错误信息
4. 尝试重新上传模板

## 📝 测试步骤

### 完整测试流程

1. **启动服务器**
   ```bash
   # 终端1 - 启动后端
   cd backend
   npm start

   # 终端2 - 启动前端
   cd frontend
   npm start
   ```

2. **上传模板**
   - 访问 http://localhost:3000
   - 点击"模板管理"
   - 点击"上传新模板"
   - 选择课表图片
   - 等待上传完成

3. **标记格子**
   - 点击"标记模式"
   - 选择星期（如"周一"）
   - 设置节次索引（如"0"）
   - 在图片上拖拽创建格子
   - 重复标记多个格子
   - 点击"保存配置"

4. **编辑课表**
   - 点击"模板列表"
   - 选择刚创建的模板
   - 点击"编辑课表"
   - 在格子中输入课程内容

5. **导出图片**
   - 点击"导出图片"按钮
   - 等待几秒钟
   - 查看浏览器控制台输出
   - 检查下载的图片

## 🔧 调试技巧

### 1. 查看详细日志

打开浏览器控制台，可以看到详细的导出过程日志：

```javascript
开始导出...
找到图片元素: <img>
图片已加载
容器元素: <div>
找到格子数量: 5
格子 0: <div> 输入框: <input>
...
```

### 2. 检查元素状态

在控制台中执行以下命令检查元素状态：

```javascript
// 检查编辑器容器
document.querySelector('.template-editor')

// 检查图片
document.querySelector('.template-editor img')

// 检查格子
document.querySelectorAll('.template-editor .cell')
```

### 3. 手动测试 html2canvas

在控制台中执行：

```javascript
const container = document.querySelector('.template-editor');
html2canvas(container).then(canvas => {
  console.log('Canvas:', canvas);
  document.body.appendChild(canvas);
});
```

## 💡 最佳实践

1. **使用现代浏览器**
   - Chrome 90+
   - Firefox 88+
   - Edge 90+

2. **确保网络稳定**
   - 使用本地网络
   - 避免使用代理

3. **使用合适的图片**
   - 建议使用 PNG 或 JPG 格式
   - 图片大小建议不超过 5MB
   - 分辨率建议在 1000x1000 以内

4. **定期保存**
   - 标记格子后及时保存配置
   - 编辑课表后及时导出

5. **清理缓存**
   - 遇到问题时尝试清除浏览器缓存
   - 使用无痕模式测试
