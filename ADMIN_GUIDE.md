# 轻量级管理员密码验证系统使用指南

## 📋 概述

本系统采用**无需注册登录**的轻量级权限管理方案，通过预设的管理员密码来区分普通用户和管理员功能。

---

## 🔑 核心特性

### ✅ 优点
- **零门槛使用**：普通用户打开网站即可直接使用所有公开功能
- **简单安全**：密码存储在服务器环境变量中，前端无法获取
- **易于配置**：只需修改 `.env` 文件即可设置密码
- **持久化状态**：管理员登录后状态保存在浏览器本地，刷新页面不丢失

### 🎯 权限划分

| 功能 | 普通用户 | 管理员 |
|------|---------|--------|
| 浏览模板列表 | ✅ | ✅ |
| 编辑课表内容 | ✅ | ✅ |
| 导出课表图片 | ✅ | ✅ |
| 风格生成器 | ✅ | ✅ |
| **上传新模板** | ❌ | ✅ |
| **标记格子位置** | ❌ | ✅ |
| **删除模板** | ❌ | ✅ |
| **保存模板配置** | ❌ | ✅ |

## ⚙️ 配置步骤

### 1. 设置管理员密码

编辑 `backend/.env` 文件，找到以下行：

```bash
# 管理员密码配置（轻量级权限管理）
ADMIN_PASSWORD=admin123456
```

将 `admin123456` 替换为你想要设置的密码。

**注意：**
- 如果不设置密码（留空或删除该行），任何人都可以进入管理模式（仅用于开发测试）
- 生产环境务必设置强密码

### 2. 重启后端服务

```bash
cd backend
npm start
```

控制台会显示：
```
Admin password configured: yes
```

## 🚀 使用流程

### 普通用户（无需密码）

1. 打开网站 `http://localhost:3000`
2. 直接浏览模板、编辑课表、导出图片
3. 右上角显示 **"🔑 管理模式"** 按钮（点击可输入密码切换为管理员）

### 管理员（需要密码）

1. 点击右上角的 **"🔑 管理模式"** 按钮
2. 在弹出的对话框中输入管理员密码
3. 点击 **"确认"** 按钮
4. 验证成功后：
   - 右上角显示 **"🔐 管理员模式"** 绿色标识
   - 导航栏出现 **"管理"** 链接
   - 模板列表页显示 **"➕ 新建模板"** 按钮
   - 可以访问 `/manager` 路由进行模板管理

### 退出管理员模式

点击 **"退出管理"** 红色按钮，立即恢复为普通用户身份。

## 🔒 安全说明

### 当前实现的安全级别

✅ **已实现：**
- 密码存储在服务器端环境变量中
- 前端通过 API 验证密码，不会暴露密码明文
- 管理员状态存储在 `localStorage` 中，仅限当前浏览器

⚠️ **注意事项：**
- 这是**前端控制**的权限系统，适合小型项目和个人使用
- 如需更高安全性，建议在后端关键 API（如上传、删除）也添加密码验证
- `localStorage` 中的数据可能被用户手动修改，但不会影响服务器端安全

### 增强安全性（可选）

如果需要在后端 API 层面也进行验证，可以在关键路由中添加中间件：

```javascript
// 示例：保护上传模板接口
app.post('/api/upload-template', (req, res) => {
  const { adminPassword } = req.headers;
  
  if (ADMIN_PASSWORD && adminPassword !== ADMIN_PASSWORD) {
    return res.status(403).json({ error: '需要管理员权限' });
  }
  
  // ... 原有逻辑
});
```

前端调用时添加 header：
```javascript
axios.post('/api/upload-template', formData, {
  headers: {
    'admin-password': localStorage.getItem('isAdmin') === 'true' ? ADMIN_PASSWORD : ''
  }
});
```

## 🛠️ 技术实现细节

### 后端 API

#### `POST /api/admin/verify`
验证管理员密码

**请求体：**
```json
{
  "password": "admin123456"
}
```

**成功响应：**
```json
{
  "success": true,
  "message": "验证成功，管理员模式已启用"
}
```

**失败响应：**
```json
{
  "success": false,
  "error": "密码错误"
}
```

#### `GET /api/admin/config`
检查是否配置了管理员密码

**响应：**
```json
{
  "success": true,
  "hasPassword": true
}
```

### 前端组件

#### `AdminModeToggle.js`
管理员模式切换组件，提供：
- 密码输入模态框
- 管理员状态管理（localStorage）
- 视觉反馈（图标和颜色变化）

#### 路由保护
- `/manager` 路由：非管理员访问时显示 "访问拒绝" 页面
- 导航栏 "管理" 链接：仅管理员可见

## 📝 常见问题

### Q1: 忘记密码怎么办？
A: 直接修改 `backend/.env` 文件中的 `ADMIN_PASSWORD` 值，然后重启后端服务。

### Q2: 可以让多人使用不同的管理员密码吗？
A: 当前版本只支持单一密码。如需多用户支持，建议集成 Supabase Auth 或 JWT 认证系统。

### Q3: 管理员状态会在什么时候失效？
A: 
- 用户主动点击 "退出管理"
- 清除浏览器缓存/本地存储
- 更换浏览器或设备

### Q4: 这个方案适合生产环境吗？
A: 
- ✅ 适合：个人项目、小型团队、内部工具
- ❌ 不适合：需要审计日志、多用户权限分级、高安全性要求的场景
